This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This repository contains the website for Ines Rzanny (rzanny.de), a German moderator, voice actress, and coach. The site is a static HTML website with two distinct versions:

1. **Legacy site** (root directory): Old version built with Adobe GoLive, using outdated HTML4 practices
2. **Current site** (Rzanny.de/): Modern version built with Apple iWeb 3.0.4, using XHTML 1.0 Transitional

## Architecture

### Site Structure

**Current Site (Rzanny.de/)**
- `index.html` - Redirects to Home.html
- `Home.html` - Main landing page showcasing services (Moderatorin, Sprecherin, Schauspielerin, Coach)
- `Vita.html` - Biography/CV page
- `Fotos.html` - Photo gallery
- `Referenzen.html` - References/portfolio
- `Coaching.html` - Coaching services (Körpersprache, Präsentation, Vorstellungsgespräch)
- `Kontakt.html` - Contact information
- `Audio_Video.html` - Audio/video samples (voice demos)

**Legacy Site (root)**
- `index.htm` - Old homepage with embedded audio (Grover Washington Jr. track)
- Various photo gallery pages (business.html, fernblick.html, ladyinred.html, etc.)
- Old navigation system using Adobe GoLive CSScriptLib.js

### JavaScript Architecture

The current site uses iWeb-generated JavaScript:
- `Scripts/iWebSite.js` - Core site functionality
- `Scripts/iWebImage.js` - Image handling
- `Scripts/Widgets/SharedResources/WidgetCommon.js` - Shared widget code
- `Scripts/Widgets/Navbar/navbar.js` - Navigation bar widget
- Page-specific JS files in `*_files/` directories (e.g., `Home_files/Home.js`)

### CSS Structure

Each page has its own CSS directory with three files:
- `[PageName].css` - Main styles
- `[PageName]IE.css` - IE7 and below compatibility
- `[PageName]Moz.css` - Mozilla-specific styles
- `Media/IE8.css` - IE8+ compatibility

### Media Files

- `Media/` - Contains MP3/M4A audio files (voice demos: AJ's Fitness, BusyLines, Bareso, Netcom, Sommerkatalog)
- Various JPG images for page content (photos, headers)
- `.mov` video file in root

## Development Notes

### Browser Compatibility

The site targets legacy browsers:
- IE7 emulation mode via meta tag: `<meta http-equiv="X-UA-Compatible" content="IE=EmulateIE7" />`
- Conditional CSS for IE7, IE8+, and Mozilla browsers
- Fixed viewport width of 700px

### Content Language

All content is in German. The site serves German-speaking clients in the media industry.

### iWeb-Generated Code

The current site is built with Apple iWeb 3.0.4. Key characteristics:
- Inline styles are heavily used
- Absolute positioning for layout
- Pages generated as XHTML 1.0 Transitional
- Navigation via navbar.js widget system
- Each page has a companion directory (`*_files/`) with page-specific assets

### Server Configuration

- `.htaccess` in root enables spell checking: `CheckSpelling on`
- CGI-bin directory with password protection for sysinfo
- Usage statistics directory with access control

## Important Constraints

1. **Do not regenerate iWeb pages** - The HTML is auto-generated by iWeb. Manual edits will be lost if pages are regenerated in iWeb.
2. **Image paths** - Images are referenced relative to page locations; maintain directory structure.
3. **Legacy code** - The root directory contains old Adobe GoLive code with deprecated HTML practices (spacer tags, inline scripts). This is intentionally preserved.
4. **Character encoding** - Site uses UTF-8 (current) and ISO-8859-1 (legacy). Maintain encoding when editing.

## Deployment

This appears to be a simple static site deployment (likely FTP to web host). No build process or package manager is used.
